# Incident Response Plan

## Severity Levels

- Critical: confirmed unauthorized access to production systems, signing credentials, user data, or payment-related systems.
- High: suspected exposure, credential compromise, or security weakness with likely user impact.
- Medium: limited issue with no confirmed user data exposure.
- Low: minor issue, policy deviation, or security improvement item.

## Response Steps

1. Identify the issue, reporter, time, and affected systems.
2. Contain by revoking access, rotating credentials, disabling affected service, or pausing release.
3. Assess whether user data, source code, certificates, App Store systems, or vendors are affected.
4. Preserve evidence and decisions.
5. Notify users, regulators, Apple, vendors, or counsel when required.
6. Remediate, verify, and document prevention actions.

## Owners

Assign an incident owner, communications owner, engineering owner, and legal/privacy reviewer for each material incident.

## Evidence

Record timeline, impact, actions, approvals, notifications, and post-incident review.
