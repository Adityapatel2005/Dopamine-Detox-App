# Privacy Rights Request Process

This process covers requests under GDPR, California privacy laws, and similar rules.

## Intake

The public release must publish a support email or web form for privacy requests. Requests should be categorized as access, deletion, correction, portability, opt-out, restriction, objection, or general privacy question.

## Identity Verification

Verify the requester using the least data necessary. For local-only data, explain that Lockd does not possess server-side app data and provide instructions for deleting local device data.

## Response Timing

- GDPR: acknowledge promptly and target a response within one month.
- California: confirm receipt within the required period and target a response within the required statutory window.
- If a request is complex, document the reason and any extension allowed by law.

## Local Data Guidance

For v1, most Lockd data is stored on device. The user can remove it by using any in-app reset control once built, deleting the app, or removing app data through iOS settings where available.

## Escalation

Escalate requests involving legal complaints, minors, sensitive data, subpoenas, security incidents, or suspected account compromise to counsel and the incident owner.

## Evidence

Record request date, request type, action taken, response date, and responsible owner in the compliance evidence log.
